# apple.com 首页拓扑

采集时间：2026-04-13  
采集方式：手工提取版，参考 `ai-website-cloner-template` 的 `clone-website` skill 与 `INSPECTION_GUIDE.md`，未使用浏览器 MCP 自动化。

## 页面段落顺序

1. 全局导航
2. 顶部轻提示带
3. 第一主 Hero
4. 连续的全宽产品 feature panels
5. 双列 promotional tiles
6. 娱乐内容横向区
7. 极密 footer

## 结构特征

- 顶部导航极薄，信息密度高，但视觉很轻
- 内容区不是单一大卡片堆叠，而是“全宽大段 + 次级双列 tile”交替出现
- 每一个 feature panel 都只做一件事：一个标题、一个一句话、一个或两个动作
- hero 和 feature panel 的主体都是“强标题 + 极少正文 + 大面积留白”
- 页面靠段落切换建立节奏，而不是靠很多小卡片

## 映射到考研现实雷达

- 全局导航：映射为站点主壳层，保留 `首页 / 统计 / 对比 / 匿名投稿`
- 顶部轻提示带：映射为“数据规模 / 样本来源边界 / 更新状态”
- 第一主 Hero：映射为首页唯一主动作“直接查目标”
- 连续 feature panels：映射为“风险判断 / 失败经验 / 横向对比”
- 双列 tile：映射为“热门入口 / 按学校浏览”
- footer：映射为“继续补样本 + 数据边界说明”

## 不映射的部分

- Apple 产品图片和硬件视觉资产
- 娱乐内容轮播
- 法律页脚和超密导航列表
- Apple 品牌语义与营销文案
